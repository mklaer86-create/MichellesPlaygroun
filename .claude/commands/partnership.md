Create or update a file for one of IMD's partnerships (e.g. the Brazil partnership).

Read `context/me.md` and `projects/imd/project.md` for context first.

There are two modes — figure out which one Michelle needs from her message,
or ask if it's unclear.

## Mode 1: New partnership

Ask Michelle (skip any she's already answered):
1. **Partner organization name and country?**
2. **What kind of partnership is this?** (church planting, leadership training,
   joint events, funding, exchange program, etc.)
3. **Status?** (prospective, active, paused, ended)
4. **Anything else she knows so far?** (contacts, how it started, language
   considerations, agreements already in place — she can paste notes or just
   describe it loosely)

Then:
- Read `projects/imd/partnerships/_TEMPLATE.md` for the structure.
- Create `projects/imd/partnerships/[country]-[partner-name].md` (lowercase,
  hyphenated — e.g. `brazil-igreja-vida.md`).
- Fill in whatever fields she gave you; leave the rest as clear placeholders.
- Add a first entry to the Update Log dated today, noting the partnership was
  set up.
- Add a line linking to the new file under a "Partnerships" section in
  `projects/imd/project.md` (create that section if it doesn't exist yet).

## Mode 2: Update an existing partnership

Ask which partnership if not obvious (list the files in
`projects/imd/partnerships/` if there's more than one, excluding `_TEMPLATE.md`).

Then:
- Read the existing partnership file.
- Add a new dated entry at the top of the Update Log with what she tells you.
- Update any other fields that changed (status, contacts, agreement details,
  open loops) — don't rewrite sections that are still accurate.

## Always

Confirm the file name and location before finishing, and show her exactly what
changed, so she can find it in Obsidian or GitHub. Keep her voice — warm and
direct, not corporate — in any narrative sections you write.
